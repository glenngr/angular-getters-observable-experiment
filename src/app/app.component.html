Using a getter to pipe onto an observable and manipulate data will always return a new instance every time the getter is called.

Can this cause performance issues?

Take a look at the console log. The getter that pipes onto the observable returns a new instance every change detection cycle.

The getter that just returns an observable without piping onto it is always the same instance.
